01  The headline

The code is in better shape than the toolchain around it

This is a mature, battle-scarred stack with real architecture — a shared strategy "spine," clean risk separation, and operational scar-tissue documented at every defensive gate. The architecture doc is accurate, not aspirational: the shared-spine and parity claims hold up against the actual code.

The fundamentals are disciplined — no import *, no bare except:, one TODO in 32k LOC, 1,774 tests with zero xfail. What's missing is an automated quality floor: there is no ruff / mypy / black config and no CI anywhere, so 700+ lint findings and several real float | None issues in live strategy code sit unnoticed. Fix that, harden a short list of latent OOM / config / async risks, then decompose five oversized modules — in that order.

02  What's genuinely strong

Credit where due — these are done right

Solid Real sim/live parity

marketdata/ (MarketState, OHLC, position_math), strategy/, and risk/caps.py are single-sourced and imported by both the engine and the backtester, with byte-identity parity tests. The "one brain, two bodies" claim holds at the data & accounting layer.

Solid risk/caps.py

Textbook single-source: three pure predicates, no IO, documented comparison operators, tested, three verified importers. Zero findings — exactly what the architecture claims.

Solid Operational maturity

Nearly every gate, tolerance constant, and loop guard cites the incident that motivated it. The event-array cache (versioned invalidation, concurrent-write safety, OOM-bounded memos) and the spawn-worker SourceConfig plumbing are mature and well-tested.

Solid Disciplined fundamentals

No import *, no bare except:, 1 TODO in the whole package, careful secrets handling (SSM SecureString + placeholder rejection), clean packaging and CLI entry points, and a large test suite with no rotting suppressions.

engine/9.5k LOC · 31 files

Functionally mature; problems are structural not logical. EngineRuntime should shed a SlotBuilder, a VenueIO helper, and per-loop classes; Reconciler.run (274-line method) and Router._book_fill (135 lines, 3 branches) should be split. CachedStateDAL thread-safety rests on a comment not a lock.

backtest/11k LOC · 36 files
disciplined; a few loose threads

Unusually disciplined for research code; the numpy/numba assembler and cache are excellent. Decompose hftbt_runner.py into latency/fills/routing. Real seam: polymarket.py re-implements the shared OHLC bucketer inline twice instead of calling resample_ohlc. Sim passes raw ndarray where the contract uses tuples.

cross-cutting hygienepyproject · tests · scripts · docs
the toolchain gap

The glaring gap is the complete absence of an automated quality floor — no linter, formatter, type-checker, or CI — which is exactly why hundreds of lint findings and real float|None issues in live strategy code go unnoticed. Smaller items: mixed logging stacks, duplicate dev-dependency groups, a heavy ML dep in the always-installed core set, and a scripts/+tools/ junk-drawer that belies the README's clean-split claim. Test suite itself is healthy (1,774 tests, 0 xfail, 6 skips).

05  Recommended action plan

Three independently-shippable phases — none require a redesign

Phase 1 — Quality floor

small · mechanical · highest leverage
  • Add [tool.ruff] + [tool.mypy] to pyproject.toml; run ruff format once.
  • Wire a pre-commit hook and a minimal GitHub Actions job: ruff check + mypy + pytest.
  • Consolidate the duplicate/conflicting dev-dependency groups; move scikit-learn/plotly to extras; add py.typed.
  • Why first: converts a well-written-but-ungated codebase into a defensible one and surfaces the float|None issues automatically.

Phase 2 — Latent-risk hardening

before scaling capital
  • Evict _books + per-question dicts in lockstep with evict_settled_questions.
  • Add extra='forbid' + Field bounds to risk config; fix the "cancelled" status Literal.
  • Add zero/None guards at risk.py:246 and market_state.py:256; lock or thread-confine CachedStateDAL.
  • Convert requests to (connect, read) timeout tuples; alert (not just log) on recorder data-drop + atomic parquet writes.

Phase 3 — Decomposition

sustained maintainability
  • Split the five god-modules. Highest value: extract theta_harvester's flag-gated experiments and collapse the 3-way param duplication to one source.
  • Decompose EngineRuntime and hftbt_runner along their tangled concerns.
  • Close remaining seams: route polymarket.py's two OHLC copies through resample_ohlc, fold the tuple/ndarray conversion into build_decision_inputs, bridge or rename the two event taxonomies.

Methodology: architecture doc read for orientation, then five parallel deep-dives audited the actual source per subsystem. Highest-severity, falsifiable claims (CI absence, the "cancelled" status mismatch, no [tool.*] config) were independently spot-verified.
