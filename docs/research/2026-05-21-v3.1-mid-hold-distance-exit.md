## Top-line

The new `exit_safety_d` gate is a clear win on **both corpora** in the
0.5–1.0 range. It closes held positions BEFORE BTC traverses the leg's
adverse boundary, cutting the long-hold losses on HL binaries and the
inventory bleed on PM that the legacy `edge_held` gate only catches AFTER
the bid has collapsed.

**Recommendation: ship `exit_safety_d = 1.0`** as the new default. It wins
on both corpora, has the best PM Sharpe, and is the largest threshold that
clears the PM 5%-regression bar.

safety_d stays above 1.5 for nearly the entire hold.

edge_held fires after the loss has already locked in — the safety_d
gate is the one that actually changes the loser's exit point.

## PM regression check (the 5% bar)

Baseline PM PnL: $719.92. The 5% regression floor is $683.92.

All of 0.25, 0.5, 0.75, 1.0 pass with **+38% to +44% improvement**. Only
1.5 fails, and it fails catastrophically. 1.0 has the best PM Sharpe at
+3.51 (vs 2.00 baseline) and the lowest PM max DD at $254 (vs $403).

## What the gate is doing mechanically

`safety_d = (ln(S / K_nearest_adverse) + (μ_eff − ½σ²)·τ) / (σ·√τ)`
— Itô-corrected per the v3.1 GBM machinery, signed positive when BTC is
on the winning side of the boundary. For a held YES on a binary, this
is `(ln(S/K) + drift) / σ√τ`. The gate fires when this drops below the
threshold; the EXIT IOC closes at the held leg's current bid with
`reduce_only=True`. Diagnostic kvs: `exit_reason=safety_d_below_threshold`,
`exit_safety_d=<d>`, `exit_threshold=<cfg>`.

## Tests

All 413 unit tests pass. The 4 new theta-harvester tests cover:
1. gate fires when safety_d < threshold AND bid is stale (legacy gate wouldn't fire)
2. gate holds when safety_d well above threshold
3. `exit_safety_d=0.0` preserves legacy behaviour exactly
4. diagnostic carries the 3 kv pairs (`exit_reason`, `exit_safety_d`, `exit_threshold`)

## Recommendation: ship `exit_safety_d = 1.0`

- Largest threshold that clears the PM 5%-regression bar (+44%).
- Best PM Sharpe across the sweep (3.51).
- Largest binary-regime improvement on HL while preserving middle-bucket gains.
- 50% smaller HL max-DD vs baseline ($130.5 vs $163.9).
- 37% smaller PM max-DD vs baseline ($254 vs $403).
- No new settlement-time risk; close path uses existing IOC-at-bid +
  `reduce_only` pattern.

`exit_safety_d = 1.5` is left in the sweep for completeness but should
NOT be shipped — the PM churn loop documents a clear failure mode at that
threshold.

Default ship value: `exit_safety_d=1.0` in the production v3.1 config.
