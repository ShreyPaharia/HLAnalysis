KPIs / acceptance bar

Analysis (trust gate)

  • Regenerates from recorded data via one command — no synthetic/stale inputs.
  • Every claim carries n + date-span + a sanity cross-check.
  • Coverage ≥30 expiries / ≥30 days.
  • Split-half stability: first 18d vs last 18d same sign for any edge claim.

Strategy (promote-to-paper gate)

  • Positive net PnL after realistic fees on book_source=recorded fills, walk-forward OOS (≥7d held out).
  • Daily Sharpe ≥ 1.0 OOS; max DD within desk inventory cap.
  • Split-half sign stability; survives adversarial verification (no lookahead, realistic fills, safety gates intact).
  • Capacity ≥ current desk size; deployable on existing engine surface (or infra cost stated).
