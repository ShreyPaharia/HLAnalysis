Critical — live engine ≠ YAML / backtest

config/strategy.yaml sets exit_safety_d: 1.0 under theta: for HL v31 and PM v31_pm, but the live engine never applies it.

Two failure modes stack:

  1. ThetaParams (Pydantic) omits strategy-only fields: exit_safety_d, vol_estimator, momentum_mr_*, vol_scaled_tte_*, lm_threshold, … Extra YAML keys are silently ignored (Pydantic default extra='ignore').
  2. build_theta_harvester_config() in engine/runtime.py only maps the small ThetaParams surface; it never passes those fields to ThetaHarvesterConfig.

Verified in this repo (load config/strategy.yaml → build config):

v31 exit_safety_d 0.0        # YAML says 1.0
v31 momentum_mr_enabled False
v31 vol_sampling_dt 5        # this one does wire through

Impact: Live v3.1 runs without the mid-hold exit_safety_d exit that backtests and operator docs treat as production (+$140 PnL / lower DD on PM per prior tuning notes). Backtest via v3_theta_harvester / build_v3_theta_harvester does honor exit_safety_d from grid params — live and sim can disagree on the same “prod” config.

Fix

  • Extend ThetaParams to cover every ThetaHarvesterConfig field (or generate both from one schema).
  • Map all fields in build_theta_harvester_config().
  • Add a round-trip test: load config/strategy.yaml → build config → assert parity with golden backtest JSON per slot.
  • Set model_config = ConfigDict(extra='forbid') on ThetaParams so stray YAML keys fail loudly.

Duplication (documented, still costly):

  • _kv_get, _winning_region, bucket leg logic copy-pasted between late_resolution.py and theta_harvester.py.
  • Three config paths for theta: dataclass → ThetaParams + build_theta_harvester_config (live) → build_v3_theta_harvester (50+ manual lines, backtest).

Simplification

  1. Extract hlanalysis/strategy/leg_math.py (shared + contract tests).
  2. Single factory: ThetaHarvesterConfig.from_params(dict) for registry and engine — delete manual builder at file tail.
  3. Split runtime.py → wiring / loops / PM watchdogs.

High impact — research scripts

26 run_* scripts and 32 config/tuning*.yaml files, many near-copies (run_v31_ablations_round2…5, six run_v1_cadence_hl* variants, v35/v37 runners).

Hardcoded paths to /Users/shreypaharia/... and stale worktrees (e.g. run_v31_ablations.py still points at an old worktree). Not portable for CI or other machines.

Fix: One CLI (scripts/research_run.py tune --suite …), env vars HLBT_DATA_ROOT / HLBT_REPO_ROOT, archive one-offs under scripts/archive/ after summaries land.

Medium — error handling in the trading loop

runtime.py uses broad except Exception: logger.exception(...) on adapter ingest, scan ticks, reconcile, and continuous checks. Keeps the process alive but can mask systematic bugs while slots keep trading.

Improvement: Latch slot.halted on repeated failures; distinguish network blips from logic errors.
